we need the java development kit (JDK) to run java through the cmd on windows
The extension of a java file is .java
We use javac filename.java to compile a java file
It then makes a bytecode which is: filename.class

In Java, bytecode is the highly optimized, platform-independent intermediate code generated by the Java compiler (javac) when it compiles human-readable Java source code (.javafile)
then we do java filename to finally run that file

It is similar to gcc making an exe file of a c program except that exe is machine readable code and .class is not machine readable

In OOPs we can use static methods of a class without making an object out of it, if its not a static method, then we need to make an object of the class and then call the method

class Dog{
    void bark(){
        System.out.print("Woof");
    }
}

public class Hello{
    public void main(String[] args){
        Dog d = new Dog();
        d.bark();
    }
}

here we dont have a static method, so in Main class, here Hello and in the main method, we make an object of class Dog, d, and then call the non_static method
We cant do anything in java without a class, if we want a non static method bark, we have to make it in a class dog, and then make an object of class dog in main class then call the method
